# Contributing Guidelines

Thank you for your interest in contributing to the Iris Species project.

This repository serves as a clean reference implementation for a traditional machine learning workflow using the Iris dataset. Contributions that improve clarity, correctness, reproducibility, or educational value are welcome.

## Types of Contributions

You may contribute by:

- Fixing bugs or errors
- Improving code quality or structure
- Enhancing documentation
- Adding tests or validation steps
- Improving notebooks or examples
- Suggesting improvements via issues

## Project Principles

Please keep the following principles in mind:

- Simplicity over complexity
- Reproducibility over experimentation shortcuts
- Readability and maintainability
- Minimal external dependencies
- Clear separation between data, code, and experiments

## Development Guidelines

- Follow the existing project structure
- Keep functions small and well-documented
- Use clear and descriptive variable and function names
- Avoid committing generated files or large artifacts
- Do not include credentials or sensitive data

## Dataset Usage

The Iris dataset is publicly available and widely used for educational purposes.

- Do not modify the raw dataset unless explicitly required
- Any preprocessing steps should be reproducible and documented
- Respect the original dataset license and attribution

## Submitting Changes

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes with clear, logical commits
4. Ensure code runs without errors
5. Submit a pull request with a concise description

## Pull Request Checklist

Before submitting a pull request, please ensure:

- Code runs successfully
- No unnecessary files are included
- Documentation is updated if applicable
- Changes are focused and scoped

## Code Review

All pull requests are subject to review. Maintainers may request revisions before merging to ensure quality and consistency.

---

By contributing to this repository, you agree to follow the Code of Conduct and these contributing guidelines.
