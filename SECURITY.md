# Security Policy

## Supported Versions

This repository is provided primarily for educational and reference purposes.

Security updates are applied only to the latest version on the `main` branch.
Older versions are not actively maintained.

## Reporting a Vulnerability

If you discover a security vulnerability related to this repository, please report it responsibly.

- **Do not** open a public issue for security-related reports.
- Report the issue privately to the repository maintainers.

When reporting, please include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Potential impact
- Suggested mitigation (optional)

Reports will be reviewed as soon as reasonably possible.

## Scope

This security policy applies to:
- Source code in this repository
- Project configuration files
- Dependency usage as defined in `requirements.txt`

This policy does **not** cover:
- Third-party datasets
- External platforms (e.g., Kaggle, cloud services)
- User environments or downstream usage

## Disclosure

Confirmed security issues will be addressed at the discretion of the maintainers.
Public disclosure will occur only after a fix or mitigation is available, when appropriate.

## Disclaimer

This project is provided "as is" without warranty.  
Users are responsible for reviewing and securing their own environments when using or adapting this code.
